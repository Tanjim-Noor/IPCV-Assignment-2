# Report Workspace

Draft each chapter in `sections/` and then reconcile the completed text in `final-report.md`. The section files are working sources; the final Markdown file is the assembled report used for the Word export.

## Required presentation

- Approximately 3,000-3,500 words.
- Microsoft Word output using Times New Roman, 12-point text, and 1.5 line spacing.
- Front cover with student name and ID, intake, module code and name, assignment title, due date, and lecturer.
- APA in-text citations and reference list.
- No source code in the main report; include only important extracts in the appendix when needed.

## Suggested word budget

| Section | Suggested words |
| --- | ---: |
| Abstract and introduction | 250-300 |
| Problem statement and objectives | 350-450 |
| Methods, implementation, and test plan | 900-1,050 |
| Experimental results and discussion | 700-850 |
| Critical evaluation | 500-600 |
| Conclusion and recommendations | 200-250 |

References, front matter, captions, and appendices are normally excluded from the narrative budget, but follow any later lecturer guidance.

## Assembly checklist

1. Replace all placeholders with project-specific evidence.
2. Keep terminology, experiment identifiers, metrics, and figure numbering consistent across sections.
3. Ensure every table and figure is introduced and interpreted in the text.
4. Copy the reconciled content into `final-report.md` and remove drafting prompts.
5. Check citations, word count, required headings, and the assignment compliance checklist.
6. Export the final document to `output/final-report.docx` and verify its rendered pages visually.
