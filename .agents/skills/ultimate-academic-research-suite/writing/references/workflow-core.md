## State record

Maintain a compact record throughout long work:

| Field | Content |
|---|---|
| Deliverable | Genre and requested artifact |
| Mode | Active workflow mode |
| Current phase | 1-7 |
| Inputs | Supplied drafts, sources, data, instructions, venue files |
| Verified evidence | Material checked directly |
| Unverified material | Claims or metadata awaiting confirmation |
| Decisions | User-approved scope, structure, terminology, style |
| Outputs | Current artifacts and locations |
| Open issues | Blockers, limitations, placeholders |
| Status | Planned, drafted, audited-with-issues, verified, submission-ready |

Do not treat an earlier summary as evidence. Reopen the underlying artifact when a high-impact claim, citation, or formatting decision depends on it.

## Seven phases

### Phase 1: Intake

Produce a Writing Brief. Do not begin a full paper when the question, deliverable, audience, or evidence basis is materially unclear.

### Phase 2: Evidence readiness

Produce an evidence inventory, search/completion plan, and design-fit note. Permit drafting with incomplete evidence only when gaps are marked and the user understands the limitation.

### Phase 3: Architecture

Produce an outline, word budget, and claim-evidence map. Require approval before a long full draft when restructuring later would be costly.

### Phase 4: Drafting

Draft bounded sections. Preserve versioned inputs and record placeholders. Do not rewrite untouched sections during a targeted task.

### Phase 5: Audit

Audit citations, claims, research integrity, style, and required disclosures. Separate errors from warnings and preferences.

### Phase 6: Review and revision

Review independently, then revise through a ledger. Repeat once when material issues remain. Do not hide unresolved comments.

### Phase 7: Finalization

Apply the current venue requirements, render the final artifact, and check the submission package.

## Checkpoints

Use checkpoints when a decision changes substantial downstream work:

1. Confirm the Writing Brief before a new full project.
2. Confirm the outline and contribution claim before full drafting.
3. Block verification when citations, data, results, or methods cannot be supported.
4. Confirm material changes to conclusions, scope, or author position.
5. Confirm the target venue before final formatting.

Do not force checkpoints for small edits, isolated abstracts, or clearly bounded formatting.

## Integrity statuses

- `PASS`: required evidence and consistency checks succeeded.
- `PASS-WITH-WARNINGS`: no critical defect; warnings remain visible.
- `FAIL`: one or more critical defects prevent verification.
- `INSUFFICIENT-EVIDENCE`: a conclusion cannot be reached from available material.

After repeated failure, the user may choose to continue drafting, but the status remains failed or insufficient and the unresolved item must appear in limitations. Never convert user acceptance into verification.

## Recovery

| Failure | Response |
|---|---|
| Ambiguous objective | Return to intake and isolate the decision |
| Missing evidence | Build a completion plan or narrow the claim |
| Conflicting sources | Record the conflict, assess source quality, avoid false reconciliation |
| Context/version drift | Identify the canonical artifact and reconcile versions before editing |
| Citation mismatch | Stop finalization and rebuild the citation-reference map |
| Review overload | Triage by criticality and group dependent changes |
| Revision degrades quality | Restore the last verified version and apply smaller patches |
| Tool unavailable | Use a portable fallback and state what could not be validated |
| Venue rule uncertain | Verify the current official instruction; do not guess |

## Completion

Completion requires:

- Requested artifact exists.
- Claims match available evidence.
- Citations and references reconcile.
- Critical integrity findings are resolved.
- Required formatting and submission components are present.
- Rendered output has been inspected when layout matters.
- Remaining limitations and warnings are explicit.
